# 贡献指南
我们欢迎并感谢所有形式的贡献！

# 提交问题
如果您发现了任何问题或错误，请在我们的问题跟踪器中提交一个问题。在提交问题时，请包括以下信息：
- 问题的描述
- 复现问题的步骤
- 预期的行为
- 实际的行为
- 相关的日志或错误消息

# 提交拉取请求
如果您想为项目做出贡献，请提交一个拉取请求。在提交拉取请求时，请包括以下信息：
- 拉取请求的描述
- 相关的问题或功能请求
- 您的代码更改
- 测试用例（如果适用）
- 文档更新（如果适用）

# 代码规范
请确保您的代码符合项目的代码规范。如果项目使用了特定的代码规范（如ESLint配置），请在提交拉取请求之前运行代码检查工具。

# 代码风格
请使用项目中定义的代码风格。如果项目没有定义特定的代码风格，请遵循常见的JavaScript编码实践。

# 代码质量
请确保您的代码质量符合项目的要求。这包括但不限于：
- 代码的可读性和可维护性
- 代码的性能和效率
- 代码的安全性

# 代码测试
请确保您的代码通过了所有的测试用例。如果项目使用了测试框架（如Jest、Mocha等），请在提交拉取请求之前运行测试。

# 文档更新
如果您对项目的文档进行了更改，请确保在提交拉取请求时更新相关的文档。这包括但不限于：
- 更新README.md文件
- 更新API文档
- 更新用户指南
- 更新贡献指南

# 更新 CHANGELOG.md 文件
如果您对项目的更改进行了记录，请确保在提交拉取请求时更新 CHANGELOG.md 文件。这包括但不限于：
- 添加新的功能
- 修复错误
- 改进性能
- 文档更新
